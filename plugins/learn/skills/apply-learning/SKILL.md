# Apply Learning

## Inputs

- `learning_proposal.md` generated by `/learn`, `/forget`, or `/consolidate`.
- Selection mode: everything, classifications, item IDs, or interactive prompts.
- Repository root, defaulting to the current working directory.

## Output

Apply selected proposal items, write backups under `.codex/learn/backups/`, update `.codex/learn/state.json`, and stage changed files with `git add`.

## Workflow

Apply interactively:

```bash
python3 ../../scripts/learn.py apply-learning --root . --proposal learning_proposal.md
```

Apply all selected proposal items non-interactively:

```bash
python3 ../../scripts/learn.py apply-learning --root . --proposal learning_proposal.md --all --yes
```

Apply only rules:

```bash
python3 ../../scripts/learn.py apply-learning --root . --proposal learning_proposal.md --classification Rule
```

Apply individual IDs:

```bash
python3 ../../scripts/learn.py apply-learning --root . --proposal learning_proposal.md --item L-abc123def456
```

## Safety

- Check `git status --short --branch` before editing.
- Warn and stop on dirty worktrees unless the user explicitly allows dirty application.
- Back up every existing target file before mutation.
- Never apply ignored observations.
- Stage only files changed by selected proposal items.
